# Mean Color at the Edge

This application demonstrates a compact environmental-imaging workflow on a Sage edge node. It acquires one frame from a node-connected camera, computes the spatial mean of each of the frame's three color channels, publishes those three measurements, and uploads the source frame with the same timestamp. The result is a small, traceable data product that links a quantitative summary to the observation from which it was calculated.

For an image with height *H* and width *W*, the application calculates each channel mean by summing that channel's pixel intensities and dividing by *H × W*. It publishes the results as `color.mean.r`, `color.mean.g`, and `color.mean.b`. Mean color is intentionally simple: it is inexpensive to calculate, reduces a full image to three values, and makes the complete path from sensor access through edge computation to Sage data publication easy to inspect. It can also serve as a basic signal for changes in scene illumination, sky conditions, vegetation state, camera obstruction, or other broad shifts in a camera's field of view.

The calculation is performed on the edge node rather than in a remote data center. Only the derived measurements and the selected supporting image enter the data workflow. In a production study, this pattern can reduce network use by uploading images only when a measurement or a separate model indicates an event worth reviewing. This tutorial uploads every captured frame so that users can validate the numerical result against its visual source; continuous deployments should adopt an event-driven or sampled upload policy appropriate to their scientific question and data budget.

The three measurements inherit the camera snapshot timestamp, allowing downstream queries to group them with the uploaded frame. Keeping a shared timestamp is important when observations from multiple sensors or applications are compared. The app uses the pywaggle `Plugin` interface for publication and upload, and the pywaggle `Camera` interface for camera access. This tutorial release selects the Sage node camera alias `left`.

Mean color is not a calibrated estimate of illumination, reflectance, atmospheric state, or ecological condition. The values can change with exposure, white balance, camera response, compression, shadows, and the scene composition. Scientific use therefore requires a stable camera configuration, quality control, suitable temporal aggregation, and validation against independent measurements. The application is best understood as a transparent edge-computing example and as a building block for more selective environmental computer-vision workflows.
